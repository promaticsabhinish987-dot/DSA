```ts
function minCost(cost, i = 0) {

//when process all the cost return 0 to collect wihle returning
    if (i >= cost.length) {
        return 0;
    }


// return by adding the sum of its two previous
    return cost[i] + Math.min(
        minCost(cost, i + 1),
        minCost(cost, i + 2)
    );
}

function minCostClimbingStairs(cost) {
    
    console.log(minCost(cost, 0))
        console.log(minCost(cost, 1))
    return Math.min(
        minCost(cost, 0),
        minCost(cost, 1)
    );
}
console.log(
    minCostClimbingStairs([10,5,3])
);



function solve(cost, i) {

    if (i <= 1) {
        return 0;
    }

    return Math.min(
        solve(cost, i - 1) + cost[i - 1],
        solve(cost, i - 2) + cost[i - 2]
    );
}

console.log(
    solve([10,2,3], 3)
);
```

## Questions

If you're learning DP pattern-by-pattern, don't jump into 100 random LeetCode problems. Train your brain to recognize the state transition behind each family.

# 1. Prefix DP (Linear DP)

**Idea:** Answer for position `i` depends on earlier positions.

```text
dp[i] = answer up to index i
```

### Beginner

1. Climbing Stairs
2. Min Cost Climbing Stairs
3. House Robber
4. Decode Ways
5. Maximum Sum Circular Subarray

### State Thinking

```text
What is best answer ending at i?
```

---

# 2. Sorted Prefix DP

**Idea:**

Sort first.

Then solve DP on sorted order.

```text
Sort
   ↓
Build answer left → right
```

Usually:

```text
dp[i]
=
best answer ending at sorted item i
```

### Beginner

6. Longest Increasing Subsequence
7. Russian Doll Envelopes
8. Maximum Length of Pair Chain
9. Largest Divisible Subset
10. Best Team With No Conflicts

### State Thinking

```text
After sorting,
which previous element
can connect to me?
```

# 3. Two Sequence DP

**Idea**

Two strings/arrays.

State tracks positions in both.

```text
dp[i][j]
```

means:

```text
using first i chars
using first j chars
```

### Beginner

11. Longest Common Subsequence
12. Edit Distance
13. Distinct Subsequences
14. Delete Operation for Two Strings
15. Uncrossed Lines

### State Thinking

```text
At i and j

Take?
Skip first?
Skip second?
Match?
```

---

# 4. Grid DP

**Idea**

State lives on cells.

```text
dp[row][col]
```

Transition from neighbors.

# Recognition Cheat Sheet

| Pattern          | State                 |
| ---------------- | --------------------- |
| Prefix DP        | `dp[i]`               |
| Sorted Prefix DP | `dp[i]` after sorting |
| Two Sequence DP  | `dp[i][j]`            |
| Grid DP          | `dp[r][c]`            |

When you can instantly identify these four patterns from a problem statement, you'll be able to solve a large fraction of beginner-to-intermediate DP problems. The next major families after these are **Knapsack DP**, **State Compression DP**, **Interval DP**, **Tree DP**, and **Digit DP**.
